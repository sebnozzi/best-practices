# Testing

* Aim at max ROI / min TCO
* Code-coverage can be misleading
* Refactor often!
* Maintainability is king
* Isolate implementation details
* Mocking / stubbing
* Proper UI testing
  * Avoid
  * Test wiring
  * Test typical cases
  * Make them maintainable
* Unit-tests: DO NOT hit the DB!
* Brittle tests
